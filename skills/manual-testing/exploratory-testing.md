# Exploratory Testing

## What is Exploratory Testing?

Simultaneous learning, test design, and execution. Tester explores application to find defects.

### Key Characteristics
- Unscripted
- Simultaneous design and execution
- Tester learning
- Real-time decision making
- Curiosity-driven

## Exploratory Testing Approach

### 1. Charter
- Define mission
- Time-box (30-90 mins)
- Focus area
- Specific goals

### 2. Exploration
- Learn application
- Find defects
- Document findings
- Ask questions

### 3. Investigation
- Deep dive
- Root cause
- Edge cases
- Patterns

### 4. Documentation
- Issues found
- Test case notes
- Observations
- Suggestions

## Techniques

### Error Guessing
- Based on experience
- Common mistakes
- What could break

### Boundary Testing
- Edge values
- Limits
- Transitions

### User Scenarios
- Real-world usage
- Typical workflows
- User expectations

## Best Practices

✅ Document charter
✅ Time-box sessions
✅ Document findings
✅ Verify issues
✅ Share learnings
✅ Combine with scripted testing
✅ Periodical review
